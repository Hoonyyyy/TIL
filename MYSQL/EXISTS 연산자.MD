## EXISTS 연산자

```
서브쿼리가 반환되는 결과값이 있ㄴ느지 조사한다.
단지 변환된 행이 있는지 없는지만 보고 값이 있으면 참 없으면 거짓을 반환한다.

1. 한 테이블이 다른 테이블과 외래키(FK)와 같은 관계가 있을 떄 유용
2. 조건에 해당하는 ROW의 존재 유무와 이후 더 수행하지 않음(지연 평가 원리이기 때문에 성능이 좋다.)
3. 일반적으로 SELECT절까지 가지 않기에 IN에 비해 속도나 성능면에서 더 좋음
4. 반대로 조건에 맞지 않는 ROW만 추출하고 싶으면 NOT EXISTS
5. 쿼리순서 : 메인 쿼리 -> EXISTS 쿼리
```
