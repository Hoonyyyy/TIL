
## AWS 연동

```
자바 스프링 프레임워크를 사용하여 MySQL을 연동하면 데이터는 MySQL 데이터베이스에 저장됩니다.

MySQL은 관계형 데이터베이스 관리 시스템(RDBMS) 중 하나로, 데이터를 테이블 형태로 저장합니다.
스프링 프레임워크에서는 데이터베이스와 연동할 때 JDBC(Java Database Connectivity) API를 사용합니다.

JDBC API는 자바 언어에서 RDBMS와 통신할 수 있는 일반적인 인터페이스를 제공합니다.
이를 통해 스프링 프레임워크는 MySQL 데이터베이스에 데이터를 삽입, 조회, 수정, 삭제할 수 있습니다.

따라서, 스프링 프레임워크를 이용하여 MySQL을 사용할 때는 MySQL 데이터베이스에 데이터가 저장되며,
이를 JDBC API를 통해 스프링 프레임워크에서 다룰 수 있습니다.


AWS(Amazon Web Services)는 클라우드 컴퓨팅 플랫폼으로, 인터넷 상에서 컴퓨팅 리소스를 제공합니다.

AWS를 사용하면 데이터 저장뿐만 아니라, 애플리케이션을 실행하는 서버, 데이터베이스, 스토리지,
네트워크 등의 인프라를 온디맨드로 사용할 수 있습니다. 이를 통해 자원을 빠르게 확장하거나 축소할 수 있어,
애플리케이션의 확장성과 유연성을 높일 수 있습니다.

따라서, AWS는 데이터 저장뿐만 아니라 애플리케이션의 인프라 구성과 관리를 위해 필요합니다. 예를 들어,
스프링 프레임워크에서 개발한 애플리케이션을 AWS EC2(Elastic Compute Cloud) 인스턴스에 배포하고,
AWS RDS(Relational Database Service)를 사용하여 MySQL 데이터베이스를 호스팅할 수 있습니다.

또한, AWS의 다른 서비스인 S3(Simple Storage Service)를 사용하여 애플리케이션에서 사용되는 파일을 저장하고,
Elastic Load Balancer(ELB)를 사용하여 트래픽을 분산하는 등의 작업을 수행할 수 있습니다. 이렇게 AWS를 사용하면,
애플리케이션의 인프라 구성과 관리를 간편하게 할 수 있습니다.






Spring, MySQL, AWS를 연동하여 사용하는 경우는 다양합니다.

일반적으로 Spring 애플리케이션을 AWS 인프라에 배포하고, MySQL 데이터베이스를 AWS RDS에 호스팅하여 데이터를 저장합니다.
이렇게 하면 다음과 같은 이점이 있습니다.

확장성: AWS는 필요에 따라 리소스를 빠르게 확장하거나 축소할 수 있어, Spring 애플리케이션의 확장성을 높일 수 있습니다.

안정성: AWS는 높은 가용성과 내결함성을 제공하여 Spring 애플리케이션의 안정성을 높일 수 있습니다.

보안성: AWS는 다양한 보안 기능을 제공하여 Spring 애플리케이션의 보안성을 높일 수 있습니다.

비용 효율성: AWS는 사용한 만큼만 비용을 지불하므로, Spring 애플리케이션의 비용 효율성을 높일 수 있습니다.

Spring, MySQL, AWS를 연동하여 사용하는 경우에는 다음과 같은 작업을 수행할 수 있습니다.

Spring 애플리케이션을 AWS Elastic Beanstalk, AWS EC2 등의 서비스에 배포합니다.

MySQL 데이터베이스를 AWS RDS에 호스팅합니다.

Spring 애플리케이션에서 MySQL 데이터베이스를 연동하고, 데이터를 삽입, 조회, 수정, 삭제합니다.

AWS S3를 사용하여 애플리케이션에서 사용하는 파일을 저장합니다.

AWS ELB를 사용하여 트래픽을 분산하고, AWS CloudWatch를 사용하여 애플리케이션의 모니터링을 수행합니다.

위와 같이 Spring, MySQL, AWS를 연동하여 사용하면, 애플리케이션의 인프라 구성과 데이터 저장,
보안, 모니터링 등의 작업을 편리하게 수행할 수 있습니다.
```
